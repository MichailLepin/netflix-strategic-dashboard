**Step 2 — Implement (GREEN phase):**

Create `src/filter.py` with `apply_filters()`:
- Takes data dict, genres list, plans list, devices list, date_range tuple
- Returns dict with keys: users, watch, recs (filtered copies)
- Date range filter: on `watch_date` column (NOT timestamp). Guard with `if len(date_range) == 2`. Convert date objects to pd.Timestamp for comparison.
- Device filter: on `device_type` in watch. Only apply `if devices:` (empty = no filter)
- Genre filter: requires join through movies table. Find movie_ids matching selected genres, then filter watch by those movie_ids. Only apply `if genres:`
- Subscription filter: filter users by `subscription_plan`, then cascade to watch and recs by user_id. Only apply `if plans:`
- All filters default to passthrough when empty list/full range

Create `src/kpis.py` with `compute_kpis()` and `get_alarm_level()`:
- `compute_kpis(filtered)` takes dict with keys users, watch, recs
- KPI-01: `watch["watch_duration_minutes"].mean() / 60.0`
- KPI-02: `(watch["progress_percentage"] >= 90).sum() / len(watch) * 100`
- KPI-03: `recs["was_clicked"].sum() / len(recs) * 100` — CRITICAL: column is `was_clicked` NOT `clicked`
- KPI-04: `(~users["is_active"]).sum() / len(users) * 100`
- ALL guard with `if len(df) > 0 else 0.0` to prevent ZeroDivisionError
- Round all to 1 decimal place

- `get_alarm_level(kpi_name, value)` returns "green"/"yellow"/"red" for CSS coloring:
  - churn_rate: green < 15, yellow 15-25, red > 25
  - completion_rate: green > 60, yellow 40-60, red < 40
  - rec_ctr: green > 20, yellow 10-20, red < 10
  - avg_watch_hours: green > 2, yellow 1-2, red < 1

Run tests — they should PASS (GREEN).

**Note:** The `apply_filters` function needs access to `data["movies"]` for the genre join, so it takes the full `data` dict but only returns the 3 filtered DataFrames (users, watch, recs).
